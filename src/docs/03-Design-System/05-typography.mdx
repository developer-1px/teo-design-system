# Typography System

우리 디자인 시스템의 타이포그래피는 **가독성(Readability)**과 **계층 구조(Hierarchy)**를 명확히 하는 것을 목표로 합니다.
`vars.css.ts`에 정의된 토큰들은 `fontSize`, `weight`, `lineHeight`, `letterSpacing`의 조합으로 사용됩니다.

## Concepts & Guidelines

### 1. Font Size (Scale)
시스템은 명확한 계층을 위해 8단계의 사이즈 스케일을 제공합니다.
*   **System UI**: `xs(11px)`, `sm(12px)`, `md(13px)`
*   **Content**: `lg(14px)`, `xl(16px)`
*   **Headings**: `xxl(20px)`, `3xl(24px)`, `4xl(32px)`

### 2. Line Height (Rhythm)
텍스트의 성격에 따라 줄 간격을 다르게 적용하여 가독성을 높입니다.
*   **none (1)**: 아이콘과 함께 쓰이는 숫자, 단일 행 텍스트
*   **tight (1.25)**: 제목(Headings), 짧은 레이블
*   **snug (1.375)**: UI 컴포넌트 내부 텍스트
*   **standard (1.5)**: 본문(Body) 문단
*   **relaxed (1.625)**: 긴 글 읽기 모드

### 3. Letter Spacing (Tracking)
*   **tighter**: 매우 큰 제목
*   **tight**: 일반 제목
*   **normal**: 본문
*   **wide**: 소제목, 강조 텍스트 (Uppercase와 함께 사용 권장)
*   **widest**: 장식적 요소

## Usage Patterns

### Application UI (System Panels)
복잡한 패널이나 탭 메뉴 등의 **System UI**는 `sm` 사이즈와 `tight` 줄 간격을 사용하여 밀도 있게 구성합니다.

```typescript
import { style } from '@vanilla-extract/css';
import { typography } from '../../styles/utils.css';

export const paletteLabel = style({
    ...typography({
        size: 'sm',
        weight: 'medium',
        height: 'tight', // 공간 절약을 위해 타이트하게
        spacing: 'normal'
    }),
    color: vars.color.gray600
});
```

### Content / Headings
문서의 본문이나 큰 제목은 읽기 편하도록 여유 있는 공간을 줍니다.

```typescript
export const heroTitle = style({
    ...typography({
        size: '4xl',
        weight: 'bold',
        height: 'tight',
        spacing: 'tight' // 큰 글씨는 자간을 좁혀 응집력 있게
    })
});

export const articleBody = style({
    ...typography({
        size: 'lg',
        height: 'relaxed' // 긴 글은 줄간격을 넓게
    })
});
```
