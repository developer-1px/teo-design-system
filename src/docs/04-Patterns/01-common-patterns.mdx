# Common Patterns

Based on the development of the Admin, Mail, and Slides applications, we have identified several recurring UI patterns. These should be standardized into reusable components to ensure consistency and speed up future development.

## 1. PageShell (Layout)

A standardized application shell that manages the high-level layout of specific tool-like interfaces.

- **Current Usage**:
  - `AdminLayout.tsx`: Sidebar + Header + Content
  - `MailPage.tsx`: Sidebar + Header + Content
  - `EditorLayout.tsx`: Sidebar + Canvas + Properties
- **Proposal**:
  - Create a `<PageShell>` component.
  - **Slots**: `sidebar`, `header`, `content`, `rightPanel` (optional).
  - **Features**:
    - Automatic handling of full-height flex layouts.
    - Integrated responsive controls (e.g., mobile sidebar toggle).
    - Consistent background colors (`surface.sunken` vs `surface.base`).

## 2. SmartTable (Data Grid)

A high-density, interactive table component for displaying collection data.

- **Current Usage**:
  - `DataTable.tsx` (Admin): User management with selection and status.
  - `EmailList.tsx` (Mail): Email rows with selection and hover actions.
- **Proposal**:
  - Create a `<SmartTable>` or `<DataGrid>` component.
  - **Features**:
    - **Selection**: Checkbox column with "Select All" support.
    - **Density**: Configurable row height (Compact, Standard, Relaxed).
    - **Sticky Header**: Keep column names visible on scroll.
    - **Custom Actions**: Row-level actions (Edit, Delete) vs Bulk actions.
    - **Status Support**: Built-in support for status rendering.

## 3. SearchFilterBar

A complex input component that handles both text search and structured filter tags.

- **Current Usage**:
  - `SmartFilter.tsx` (Admin): Tag-based filtering (Status: Active).
  - `Header.tsx` (Mail): Search bar with "tune" options.
- **Proposal**:
  - Create a `<SearchFilterBar>` input.
  - **Features**:
    - **Tokenized Inputs**: Pressing Enter creates a search chip/tag.
    - **Autocomplete**: Suggest keys (e.g., `is:`, `from:`) and values.
    - **Clearable**: Easy way to reset filters.
    - **Visuals**: Unified look similar to the admin filter bar.

## 4. ResizablePanel

A layout primitive that allows users to adjust the width of sidebars or panels.

- **Current Usage**:
  - `useResizable.ts` (Editor): Custom hook for resizing the left sidebar.
  - *Wanted in*: Mail sidebar, Admin detail drawer.
- **Proposal**:
  - Create a `<ResizablePanel>` component or hook library.
  - **Features**:
    - Drag handle styling (invisible usually, visible on hover/active).
    - Min/Max width constraints.
    - `onResizeEnd` callback for persisting preference to local storage.

## 5. StatusBadge

A visual indicator for state (Active, Offline, Error, etc.).

- **Current Usage**:
  - `DataTable.tsx` (Admin): Shows User status (Active/Pending).
  - *Wanted in*: Mail (labels), Editor (connection status).
- **Proposal**:
  - Create a `<StatusBadge>` atom.
  - **Variants**: `neutral`, `success`, `warning`, `critical`, `info`.
  - **Design**: Small dot + Text label, or pill shape.

## 6. Toolbar

A horizontal or vertical container for action buttons.

- **Current Usage**:
  - `Header.tsx` (Mail): Actions like Settings, Support.
  - `EditorSurface.tsx` (Editor): Formatting tools.
- **Proposal**:
  - Create `<Toolbar>` and `<Toolbar.Item>` components.
  - **Features**:
    - Consistent spacing and sizing.
    - Built-in tooltip support for items.
    - Divider support.
    - Toggle state support (pressed/active).

## 7. Modal / Drawer

Overlay surfaces for secondary tasks.

- **Current Usage**:
  - `Admin`: Needs "Detail Drawer" for editing users.
  - `Mail`: Needs "Compose" modal.
- **Proposal**:
  - Create `<Drawer>` (side sheet) and `<Modal>` (center dialog) sharing a common `<Overlay>` primitive.
  - **Features**:
    - Backdrop blur/dimming.
    - Focus trapping.
    - Esc key close.
    - Animations (slide-in / fade-in).
