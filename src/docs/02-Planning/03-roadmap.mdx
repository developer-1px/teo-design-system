# Design System Roadmap

현재 디자인 시스템의 구현 상태와 앞으로의 계획입니다.

## 1. 완료된 기능 (Implemented)

### Foundation (Tier 1)
- [x] **Surface System**
  - **Hybrid Approach**: `surface.base` (Static) vs `surface.interactive` (Dynamic Mixin)
  - `vars.surface` 토큰화 및 Light/Dark 모드 지원
- [x] **Semantic Color System**
  - **Zinc Palette Mapping**: 모든 컬러를 Zinc 스케일로 통일
  - **Semantic Aliases**: `primary`, `secondary`, `destructive`, `muted`, `accent` 등의 의미론적 토큰 도입
- [x] **Typography System**
  - **Refactor**: 순수 타이포그래피 스케일 (`textStyle`)과 문서 레이아웃 (`.prose`) 분리
  - `vars.fontSize`, `vars.weight`, `vars.lineHeight` 토큰화

### Layout Architecture (Pro Shell)
- [x] **Shell Primitive**
  - VS Code 스타일의 "Pro Tool" Grid 레이아웃 (Navbar, Sidebar, AuxPanel, BottomPanel 등 슬롯 지원)
- [x] **Layout Primitives**
  - **Panel**: 공용 사이드바/패널 컴포넌트
  - **TopBar**: 표준화된 헤더 컴포넌트
- [x] **Reference Implementations**
  - **AdminLayout**: `/admin` 라우트용 쉘
  - **SlidesPage**: 복잡한 디자인 도구 레이아웃에 `Shell` 적용 완료

### Core Components
- [x] **Button System**
  - `@vanilla-extract/recipes` 기반
  - **Variants**: Solid, Ghost, Outline, Link
  - **Features**: Polymorphism (`as` prop), Loading State, Shapes (Square/Circle)
- [x] **Form Elements**
  - TextInput, Select, Checkbox, Switch, SearchBar
- [x] **Interaction Primitives**
  - **VisualFrame**: Hover/Select/Badge 공통 인터랙션 (Storyblok Editor 등에서 사용)
