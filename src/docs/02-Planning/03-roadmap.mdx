# Design System Roadmap

현재 디자인 시스템의 구현 상태와 앞으로의 계획입니다.

## 1. 완료된 기능 (Implemented)

### Foundation (Tier 1)
- [x] **Surface System**
  - Base, Subtle, Card, Highlight, Ghost 계층 정의
  - Light/Dark 모드 완벽 지원
  - `vars.surface` 토큰화 완료
- [x] **Typography System**
  - Semantic Scale (Title, Subtitle, Body, Caption)
  - `vars.fontSize`, `vars.weight`, `vars.lineHeight` 토큰화
- [x] **Layer System** (ex. Overlay)
  - **Positioning Layers**: Sticky, Fixed, Absolute
  - **Semantic Overlays**: Modal, Drawer, Tooltip
  - Z-Index & Elevation 토큰화 완료
- [x] **Interaction Primitives**
  - **VisualFrame**: Hover/Select/Badge 공통 인터랙션 추출 완료
- [x] **DevTools**
  - Vite Plugin Inspector (Meta+D)
  - Source Code Jump 기능

## 2. 진행 중 (In Progress)

### Component Library (Tier 1)
- [/] **Form Elements** (Admin UI 필수)
  - [x] **TextInput System**: Size variants (Standard, Dense), States (Error, Disabled)
  - [x] **Selection Controls**: Checkbox, Switch, Radio
  - [x] **Select/Dropdown**: Native & Custom variants
- [ ] **Data Display**
  - [ ] **Table System**: Admin List View, Density variants
  - [ ] **Status Badges**: Semantic colors (Success, Warning, etc.)

## 3. 보류됨 (On Hold)

### Layout System
> [!NOTE]
> `<Box>`, `<Stack>` 등의 추상화 컴포넌트는 디버깅 복잡도를 높일 수 있어, Vanilla Extract CSS 파일을 직접 사용하는 방식을 유지합니다. 필요시 CSS Utility 패턴만 정의합니다.

- [ ] **Layout Patterns**: Utility-first CSS (`sprinkles` 도입 여부 재검토)

## 4. 예정됨 (Backlog)

### Motion System (Tier 2)
- [ ] **Transition Tokens**
  - `vars.duration`, `vars.easing` 정의
  - 감각적인 마이크로 인터랙션 (Hover, Press, Dialog Enter/Exit)
