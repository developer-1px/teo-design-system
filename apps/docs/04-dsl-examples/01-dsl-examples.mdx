# DSL Design System

Why-First 접근법을 보여주는 실제 예시입니다.

<DSLDemo />

## DSL 시스템이란?

DSL (Domain Specific Language)은 **Purpose-Based Design** 철학을 코드로 구현한 것입니다.

개발자는 "왜"만 설명하고, 시스템이 "어떻게"를 자동으로 결정합니다.

### 기존 방식 (How-based)

```tsx
<div className="flex flex-col gap-4 p-6 bg-white rounded-lg shadow-md">
  <h1 className="text-2xl font-bold text-gray-900">제목</h1>
  <p className="text-base text-gray-600">설명</p>
  <button className="bg-blue-600 text-white px-6 py-3 rounded-lg">
    시작하기
  </button>
</div>
```

### DSL 방식 (Why-based)

```tsx
<Section prominence={1}>
  <Group purpose="content">
    <Item prominence={1}>제목</Item>
    <Item prominence={2}>설명</Item>
  </Group>

  <Group purpose="action">
    <Item prominence={1}>시작하기</Item>
  </Group>
</Section>
```

## 핵심 개념

### 1. Purpose (목적)

"이 그룹이 왜 존재하는가?"

- **navigation** - 다른 곳으로 이동
- **action** - 무언가를 실행
- **form** - 데이터를 입력
- **content** - 정보를 전달
- **list** - 여러 항목을 보여줌
- **status** - 현재 상태를 알림

### 2. Prominence (주목도)

"얼마나 중요한가?"

- **1 (Primary)** - 가장 먼저 봐야 할 것
- **2 (Secondary)** - 그 다음으로 중요한 것
- **3 (Tertiary)** - 보조적인 것

### 3. 시스템이 자동으로 하는 일

- 적절한 UI 패턴 제안
- 토큰 자동 적용 (색상, 크기, 간격)
- 시멘틱 HTML 생성
- 키보드 접근성 설정
- 일관된 시각적 계층 유지

## 예시 코드

### Hero 섹션

```tsx
<Section prominence={1}>
  <Group purpose="content" prominence={1}>
    <Item as="h1">메인 제목</Item>
    <Item as="p" prominence={2}>설명 텍스트</Item>
  </Group>

  <Group purpose="action" prominence={1}>
    <Item as="button">시작하기</Item>
    <Item as="button" prominence={2}>더 알아보기</Item>
  </Group>
</Section>
```

### 네비게이션

```tsx
<Region role="header">
  <Group purpose="navigation" direction="horizontal">
    <Item as="a" href="/" prominence={1}>홈</Item>
    <Item as="a" href="/about" prominence={2}>소개</Item>
    <Item as="a" href="/docs" prominence={2}>문서</Item>
    <Item as="a" href="/contact" prominence={3}>연락처</Item>
  </Group>
</Region>
```

### 기능 목록

```tsx
<Section prominence={2}>
  <Group purpose="content" prominence={1}>
    <Item as="h2">주요 기능</Item>
  </Group>

  <Group purpose="list" prominence={2}>
    {features.map(feature => (
      <Item key={feature.id}>
        <h3>{feature.title}</h3>
        <p prominence={3}>{feature.description}</p>
      </Item>
    ))}
  </Group>
</Section>
```

## 장점

1. **생산성 향상** - 디자인 결정을 시스템에 위임
2. **일관성** - 자동으로 일관된 UI 생성
3. **접근성** - 자동으로 ARIA 속성 및 키보드 네비게이션 설정
4. **유지보수성** - 목적 기반 코드는 이해하기 쉬움
5. **확장성** - 새로운 컴포넌트 추가가 쉬움
