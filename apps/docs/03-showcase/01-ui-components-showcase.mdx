## 컴포넌트 카테고리

### Atoms (원자)

기본적인 UI 빌딩 블록:

- **Buttons** - 다양한 크기와 스타일의 버튼
- **Badges** - 상태를 나타내는 뱃지
- **Avatars** - 사용자 아바타
- **Tags** - 제거 가능한 태그
- **Checkbox & Radio** - 선택 컨트롤
- **Switch** - 토글 스위치

### Molecules (분자)

결합된 UI 컴포넌트:

- **Alerts** - 정보/성공/경고/에러 알림
- **Search Input** - 검색 입력 필드
- **Dropdown** - 드롭다운 메뉴
- **Cards** - 다양한 카드 변형

### Forms (폼)

폼 관련 컴포넌트:

- **Form Fields** - 레이블, 도움말, 에러 메시지가 포함된 폼 필드
- **Input** - 텍스트 입력
- **Select** - 선택 드롭다운
- **Textarea** - 멀티라인 텍스트 입력

## 사용 가이드

### Button 사용법

```tsx
import { Button } from '@/components/ui/Button';

// 기본 사용
<Button variant="accent">Primary Action</Button>
<Button variant="ghost">Secondary Action</Button>
<Button variant="outline">Dangerous Action</Button>

// 크기 변형
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>
```

### FormField 사용법

```tsx
import { FormField, Input } from '@/components/ui';

<FormField
  label="Username"
  required
  helper="Enter your username"
  error={errors.username}
>
  <Input placeholder="johndoe" />
</FormField>
```

### Badge 사용법

```tsx
import { Badge } from '@/components/ui/Badge';

<Badge variant="success">Success</Badge>
<Badge variant="warning">Warning</Badge>
<Badge variant="error">Error</Badge>
<Badge variant="info">Info</Badge>
```

## 디자인 시스템 원칙

모든 컴포넌트는 다음 원칙을 따릅니다:

1. **Purpose-Based Design** - "왜" 존재하는지 명확한 목적
2. **Prominence System** - 1-3 단계의 주목도 체계
3. **7-Layer System** - 시각적 깊이 계층 구조
4. **최소 토큰** - 제한된 디자인 토큰 사용
5. **접근성 우선** - ARIA 속성 및 키보드 네비게이션
